什么是技能？

技能（Skills）是 OpenClaw 的功能扩展模块。每个技能为 AI 助手提供特定的能力，比如查询天气、发送邮件、操作文件等。

🧩 技能的核心概念

  • 工具（Tools） - AI 可以调用的功能，如搜索、计算、API 调用
  • 资源（Resources） - 技能提供的数据源，如文件、数据库、API
  • 提示词（Prompts） - 预定义的对话模板
  • 处理器（Handlers） - 处理特定指令的逻辑

技能 vs 插件

特性 传统插件 OpenClaw Skills
安装方式 手动下载安装 一行命令，自动安装
AI 理解 固定命令 自然语言调用
权限控制 全局权限 细粒度权限管理
隔离性 共享环境 沙箱隔离

ClawHub 使用

ClawHub 是 OpenClaw 的官方技能市场，汇集了社区贡献的 1715+ 个技能。

访问 ClawHub

你可以通过以下方式访问 ClawHub：

# 命令行方式
openclaw hub search     # 搜索技能
openclaw hub info     # 查看技能详情

# Web 界面
openclaw hub open                  # 在浏览器中打开 ClawHub

技能信息解读

每个技能页面包含以下信息：

  • 名称和描述 - 技能的功能说明
  • 作者 - 技能开发者信息
  • 版本 - 当前版本和更新历史
  • 下载量 - 社区使用热度
  • 评分 - 用户评价
  • 依赖 - 需要的其他技能或系统工具
  • 权限 - 需要的系统权限

💡 选择技能的建议

  • 优先选择官方认证或高下载量的技能
  • 检查最近更新时间，避免使用废弃技能
  • 仔细阅读权限要求，避免过度授权
  • 查看用户评价和 Issue 反馈

技能安装

安装技能非常简单，有多种方式可供选择。

方法 1：命令行安装

# 安装指定技能
openclaw install weather

# 安装特定版本
openclaw install weather@1.2.0

# 从 GitHub 安装
openclaw install github:username/repo

# 从本地路径安装
openclaw install ./path/to/skill

方法 2：ClawHub CLI

# 使用 npx 直接运行
npx clawhub@latest install weather

# 安装多个技能
npx clawhub@latest install weather email search

方法 3：Dashboard 界面

1. 打开 OpenClaw Dashboard（http://localhost:3000）

2. 进入「技能市场」页面

3. 搜索或浏览技能

4. 点击「安装」按钮

配置技能

部分技能需要配置才能使用。配置方式：

# 查看技能配置项
openclaw config weather

# 设置配置项
openclaw config weather.apiKey=your_api_key
openclaw config weather.defaultCity=Beijing

# 或者编辑配置文件
openclaw config edit weather

管理已安装技能

# 列出已安装技能
openclaw list

# 查看技能详情
openclaw info weather

# 更新技能
openclaw update weather

# 更新所有技能
openclaw update --all

# 卸载技能
openclaw uninstall weather

# 启用/禁用技能
openclaw enable weather
openclaw disable weather

📝 配置文件位置

  • 全局配置 - ~/.openclaw/config.json
  • 技能配置 - ~/.openclaw/skills//config.json
  • 环境变量 - 以 OPENCLAW_ 开头的环境变量

权限管理

OpenClaw 提供细粒度的权限控制，确保技能只能访问必要的资源。

权限类型

权限 说明 风险等级
read:file 读取文件 低
write:file 写入文件 中
execute:shell 执行 shell 命令 高
network:request 发起网络请求 中
memory:read 读取记忆 低
memory:write 写入记忆 中

配置权限

在 AGENTS.md 或配置文件中设置权限：

{
  "skills": {
    "weather": {
      "permissions": ["network:request"],
      "allowedHosts": ["api.weather.com", "api.openweathermap.org"]
    },
    "file-manager": {
      "permissions": ["read:file", "write:file"],
      "allowedPaths": ["/home/user/documents", "/tmp"]
    },
    "code-runner": {
      "permissions": ["execute:shell"],
      "sandbox": true,
      "timeout": 30000
    }
  }
}

运行时权限提示

当技能请求未授权的权限时，OpenClaw 会提示你确认：

⚠️ 技能 "file-manager" 请求以下权限：
  - write:file (/etc/passwd)

允许此次操作？
[Y] 是  [N] 否  [A] 始终允许  [D] 拒绝并禁用技能

🔒 权限最佳实践

  • 遵循最小权限原则，只授予必要的权限
  • 使用 allowedHosts 限制网络访问范围
  • 使用 allowedPaths 限制文件访问范围
  • 对高风险权限（如 shell 执行）启用沙箱
  • 定期审查已安装技能的权限

安全注意事项

技能可以执行代码和访问系统资源，安全使用至关重要。

⚠️ 安全提醒

近期安全研究发现，ClawHub 上存在恶意技能，可能窃取用户数据或执行恶意操作。请务必遵循以下安全准则。

技能安全检查清单

  • 检查技能作者信誉（GitHub 历史、社区评价）
  • 查看技能源代码（优先选择开源技能）
  • 审查权限要求是否合理
  • 检查依赖项是否安全
  • 在隔离环境中首次测试
  • 定期检查技能更新和安全公告

危险信号 🚩

以下情况应引起警惕：

  • 要求 execute:shell 权限但没有明确说明用途
  • 请求访问敏感路径（~/.ssh, ~/.aws, 密码文件等）
  • 请求访问未说明的远程服务器
  • 代码混淆或经过混淆打包
  • 新发布但下载量异常高（可能是刷量）
  • 作者信息缺失或可疑

安全工具

🔍 SecureClaw

OWASP 对齐的安全审计插件，可自动扫描技能中的安全风险。

openclaw install secureclaw
openclaw run secureclaw audit

🛡️ 沙箱模式

在隔离环境中运行技能，限制其对系统的影响。

openclaw config skills.sandbox=true
